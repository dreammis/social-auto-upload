## Context

当前发布中心（`PublishPage`）采用 `xl:grid-cols-3` 布局：左侧 2/3 为表单（视频/图文），右侧 1/3 为 AI Sidebar + PublishPreview + PublishOverview。右侧列信息过载，纵向拥挤。

## Goals / Non-Goals

**Goals:**
- 将 AI 内容生成面板从右侧 sidebar 迁移到底部抽屉面板
- 为 AI 生成流程提供全宽舞台，彻底解决拥挤问题
- 保持现有 AI 功能完整（模型选择、输入、生成、结果、历史、模板等）
- 平滑的展开/收起动画体验
- 用户可拖拽调整面板高度

**Non-Goals:**
- 不改 AI 生成逻辑（prompt 构建、API 调用、流式响应）
- 不新增 AI 功能（如多轮对话、图片生成）
- 不改动后端 API
- 不改表单本身的交互逻辑

## Decisions

### 1. 面板状态：三级状态机

底部面板支持三种状态：
- **`collapsed`**（收起）：只显示一个 48px 高的工具栏，包含 "AI 助手" 标签、快捷状态指示、展开按钮
- **`peek`**（半展开）：面板高度约 320px，显示输入区和生成按钮，适合快速输入 prompt
- **`expanded`**（全展开）：面板高度约 520px 或用户自定义，完整展示所有模块

状态切换通过点击工具栏或拖拽实现。

### 3. 拖拽调整高度

面板顶部边框区域（`resize-handle`）支持鼠标拖拽：
- 拖拽时面板实时跟随鼠标 Y 位置
- 最小高度 200px（`peek` 状态），最大高度视口高度 - 120px
- 释放后记忆用户设定的高度，下次展开恢复
- 使用 `react-resizable-panels` 或原生 `onMouseMove` 实现

### 4. PublishPreview / PublishOverview 重新安置

当前右侧列的两个组件：
- **`PublishPreview`**：移到表单底部，提交按钮上方，以折叠卡片形式展示
- **`PublishOverview`**：移到表单顶部，平台选择下方，以水平统计条形式展示

### 5. 动画策略

- 面板展开/收起：`framer-motion` 的 `animate={{ height }}`，duration 0.25s，ease `[0.25, 0.46, 0.45, 0.94]`
- 内容淡入：`opacity` 0→1，staggerChildren 0.05s
- 拖拽时无动画，直接跟随鼠标

## Risks / Trade-offs

- **[表单区域被遮挡]** → 面板最大高度限制为 `vh - 120px`，始终保留表单可见区域
- **[移动端体验]** → 移动端面板改为全屏 Modal，而非底部抽屉
- **[代码改动量大]** → 分两步实施：① 布局重构 ② 面板细节打磨
- **[用户习惯改变]** → 首次展开时显示轻量级引导提示

## 组件拆分建议

为降低复杂度，建议将当前 `AiSidebar.tsx`（~1050 行）拆分为：

| 组件 | 职责 | 文件 |
|---|---|---|
| `AiPanel` | 底部面板容器、状态管理、拖拽逻辑 | `AiPanel.tsx` |
| `AiPanelToolbar` | 收起状态的工具栏 | `AiPanelToolbar.tsx` |
| `AiInputArea` | 输入框、附件、模板、增强 | `AiInputArea.tsx` |
| `AiResultArea` | 生成结果展示、操作按钮 | `AiResultArea.tsx` |
| `AiSettingsBar` | 模型选择、Key 管理（内联） | `AiSettingsBar.tsx` |

## 验收标准

- [ ] 底部面板支持展开/收起/拖拽调整高度
- [ ] 表单区域占满全宽，无右侧 sidebar 占位
- [ ] AI 生成流程完整可用（输入 → 生成 → 结果 → 应用）
- [ ] 动画平滑，无闪烁或跳动
- [ ] 移动端面板以 Modal 形式正常工作
- [ ] PublishPreview 和 PublishOverview 在新布局中正常展示
