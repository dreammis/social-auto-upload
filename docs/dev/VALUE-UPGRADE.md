# 项目升值改进建议

> 原则：最少改动，最大感知价值。每一项都是用户一眼能看到的提升，不需要重写架构。
>
> **更新于 2026-06-24**：部分建议已在重遵中落地，详见下表“落地状态”列。提案类文档，与已交付 commit 并不强耦合，仅作 roadmap 参考。

## 一、已在做但可以再推一步的事

以下功能已经实现，加一点打磨即可显著提升"成品感"：

### 1. 首次使用引导（Onboarding）

**现状**：空状态已有 `EmptyState` 组件。  
**加一步**：首次打开页面时，弹一个 3 步引导浮层：

- Step 1：「先创建一个账号分组」→ 指向「新建分组」按钮
- Step 2：「登录平台账号」→ 指向分组内的授权区域
- Step 3：「去发布中心上传」→ 指向侧栏「发布中心」

**感知价值**：从「工具脚本」到「产品」。  
**工作量**：~80 行，一个 `<Dialog>` + 3 个 step 状态。  
**ponytail**：用 `localStorage` 存 `onboarding_done`，不建后端表。

### 2. 发布成功的"庆祝感"

**现状**：`PublishSuccessBanner` 已存在，发布后跳转任务列表。  
**加一步**：发布按钮点击后加一个 0.6s 的 confetti 动画（`canvas-confetti`），然后显示 banner。

```
# ponytail: single npm add, one onSuccess call, ~5 行集成
npm add canvas-confetti
```

**感知价值**：发布完看到彩带，用户会觉得"这东西做得很用心"。  
**工作量**：~10 行。

## 二、三个低投入高回报的新功能

### 3. 发布的视频/图文内容预览

**现状**：发布表单填完直接提交，用户看不到最终效果。  
**建议**：在发布表单旁加一个简单的预览面板，展示：

- 标题 + 描述/内容 + 标签 的聚合预览
- 如果是视频模式：封面图 + 标题排版
- 如果是图文模式：图片轮播 + 文字排版

**感知价值**：减少"发出去才发现标题写错了"的焦虑。  
**工作量**：~120 行，一个 `<PreviewCard>` 组件，从表单 state 读数据。  
**ponytail**：纯展示，无后端改动。已有数据都在 store 里。

### 4. 任务进度可视化

**现状**：任务列表是表格，状态是文字 Badge。  
**建议**：在任务列表顶部加一条汇总进度条：

```
████████████░░░░░░░░░░  已完成 8/12
```

每个任务卡片加一个微型进度指示器（上传中 → 处理中 → 发布中 → 完成）。

**感知价值**：批量发布时一眼看到整体进度，不需要逐行扫。  
**工作量**：~60 行，一个 `<ProgressBar>` 组件 + 状态统计。  
**ponytail**：数据已在 `useTasks` 返回的 list 里，纯前端聚合。

### 5. 暗色/亮色模式的"品牌色"一致性

**现状**：有 ThemeProvider，支持亮色/暗色/系统。  
**加一步**：给项目定一个品牌色，所有页面的 Primary 按钮、链接、高亮都统一用它。当前用的是 shadcn 默认色。

**感知价值**：从"通用模板"到"有品牌的产品"。  
**工作量**：改 4 个 CSS 变量，~5 行。  
**ponytail**：在 `index.css` 里覆盖 `--primary` 和 `--primary-foreground`。参考 `DESIGN.md` 里的 Linear 色板。

## 三、删掉的东西也能升值

### 6. 清理遗留文件

**现状**：`examples/` 下有多个旧脚本（`upload_to_douyin.py` 等），README 还引用了旧端口号 5409。  
**建议**：删除 `examples/` 下已被 CLI 替代的脚本；README 端口号已改到 6001（看 diff 已完成）。

**感知价值**：新用户不会困惑"到底用哪个入口"。  
**工作量**：删文件，~0 行新代码。

## 四、可以但不急的事

以下功能有价值，但在上面 6 项做完之前不建议开始：

- ❌ 移动端 PWA 支持 — 当前是桌面工具，移动端不是核心场景
- ❌ 多语言 i18n — 用户群是国内，中文够用
- ❌ 数据仪表盘/分析页 — 需要后端统计，投入大
- ❌ 多用户/权限系统 — 单用户桌面工具不需要

**ponytail**：这些等到有用户明确要求再做。YAGNI。

| 1 | confetti 庆祝动画 | 10 分钟 | 一行 import，感知价值极高 | ⏳ 未开始 |

| 4 | 内容预览面板 | 1 小时 | 核心体验提升 | ⏳ 未开始 |
| 5 | 任务进度可视化 | 45 分钟 | 批量场景刚需 | ⏳ 未开始 |
| 6 | 首次使用引导 | 1 小时 | 新用户体验质变 | ⚠️ `feat(onboarding): migrate to @reactour/tour spotlight + reset entry`（迁移到第三方控件，未走 3-step Dialog） |

**总计**：约 3.5 小时，6 项改进，项目从"能用"变成"好用"。
